Search options

A search query consists of a fulltext expression and optional parameters.

Fulltext expression

Following fulltext search methods are available.

"text to search"
Searches for text containing a phrase "text to search"
Example: "Ministerstvo zahraničních věcí" – will return objects with any property containing phrase "Ministerstvo zahraničních věcí".
text to search
Is the same as "text to search".
Example: Ministerstvo zahraničních věcí.
"text*"
By default, the search algorithm considers whole words only. To alter this behavior, you can include an asterisk in the query. Due to the limitations of the search engine the word prefix must be at least four characters long.
Example: "ministerstv*" – will return objects with any property containing a word that starts with "ministerstv", i. e. "ministerstvo" or "ministerstva".
"phrase" AND "other"
"phrase" OR "other"
Logical conjunction and disjunction
Example: "Česká republika" AND "Praha" – will return objects with any property containing both phrases "Česká republika" and "Praha".

Note: Some fulltext search methods can have significant impact on performance. When using the asterisk or logical disjunction, it is recommended to narrow the data domain to search at, for example by introducing type or property restrictions described below.

Search parameters

To refine the search, you can specify any number of following search parameters.

type:URI
Constrains the algorithm so that it searches in properties of objects of specified RDF type.
Example: "trestní zákoník" type:lex:Act – will return objects of type lex:Act with any property containing text "trestní zákoník".
property:URI
Specifies a particular property of an object to search in
Example: 2015 property:dcterms:title – will return objects that have a property dcterms:title containing text "2015".
graph:URI
Limits the query to a particular graph
Example: Praha graph:http://linked.opendata.cz/resource/dataset/seznam.gov.cz/ovm – will return objects that belong to the specified graph (or dataset) and that have any property that contains text "Praha".

Note: You can use both prefixed and not prefixed variants of URIs interchangeably.
